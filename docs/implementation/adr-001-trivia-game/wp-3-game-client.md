# WP-3: Game Client

**Depends on**: WP-2 (working WebSocket protocol to connect to)
**Enables**: WP-6 (Polish)

## Goal

Build the single-file HTML/CSS/JS game client that runs inside Telegram's WebView. After this WP, the game is fully playable with a polished mobile-first UI.

---

## Issues

### 3.1 — Client foundation & WebSocket connection

**Type**: Feature
**Effort**: Medium

Set up the HTML structure, CSS reset, WebSocket connection logic, and the screen management system.

**Acceptance criteria**:
- Single `index.html` file with all CSS and JS inline
- Connects to WebSocket server on load (derives WS URL from page URL)
- Screen system: only one screen visible at a time, transitions between them
- Screens defined: `lobby`, `pregame`, `question`, `reveal`, `leaderboard`, `podium`, `error`
- Auto-reconnect on WebSocket drop (3 attempts, 2s apart) with state recovery
- Reads room code from URL query param (`?room=abc123`)
- Works without Telegram WebApp API (fallback for browser testing — generates a random player name)

**Deliverables**:
- `apps/trivia/index.html` with foundation code

---

### 3.2 — Lobby screen

**Type**: Feature
**Effort**: Small

The waiting room where players see who's joined before the game starts.

**Acceptance criteria**:
- Shows room code prominently (for sharing)
- Live-updating player list with names (and photos if available from Telegram)
- Player count: "3 players joined"
- Room creator sees a "Start Game" button (enabled when 2+ players)
- Non-creators see "Waiting for host to start..."
- Smooth entry animation when a new player joins

**Deliverables**:
- Lobby screen in `index.html`

---

### 3.3 — Question screen

**Type**: Feature
**Effort**: Medium

The core gameplay screen — question text, 4 answer buttons, countdown timer.

**Acceptance criteria**:
- Question text displayed prominently at top
- Question counter: "3 / 7"
- 4 answer buttons in a 2x2 grid, Kahoot-style colors (red, blue, green, orange)
- Buttons are large touch targets (minimum 60px height, full width on mobile)
- Countdown bar animates from full to empty over the time limit
- Tapping an answer: button visually locks in (highlight), other buttons grey out, no double-tap
- If timer expires without answer: auto-transition to reveal screen
- Haptic feedback on tap (via Telegram WebApp API, if available)

**Deliverables**:
- Question screen in `index.html`

---

### 3.4 — Answer reveal screen

**Type**: Feature
**Effort**: Small

Shows the correct answer, the player's result, fun fact, and Atlas commentary.

**Acceptance criteria**:
- Correct answer highlighted in green, wrong answers greyed out
- Player's selected answer marked (checkmark if correct, X if wrong)
- Points earned this round displayed with animation: "+1350"
- Streak indicator if on a streak: "3 in a row!"
- Fun fact text displayed below the question
- Atlas commentary displayed (styled differently — like a chat bubble or quote)
- Auto-transitions to leaderboard after 5 seconds

**Deliverables**:
- Reveal screen in `index.html`

### 3.6 — Podium / game over screen

**Type**: Feature
**Effort**: Medium

Final celebration screen after the last question.

**Acceptance criteria**:
- Top 3 players displayed as a podium (2nd | 1st | 3rd layout, 1st elevated)
- Player names, scores, and position numbers
- CSS confetti or particle effect (no external library — CSS keyframes or small inline JS)
- "Your rank: #X" for non-podium players
- Atlas summary text (generated by AI or template fallback)
- "Close" button that calls `Telegram.WebApp.close()` (or just closes tab in browser)

**Deliverables**:
- Podium screen in `index.html`

---

### 3.7 — Telegram Web App API integration

**Type**: Feature
**Effort**: Small

Wire up Telegram-specific APIs for identity, theming, and haptics.

**Acceptance criteria**:
- Player name and ID sourced from `Telegram.WebApp.initDataUnsafe.user` when available
- Theme colors applied from `Telegram.WebApp.themeParams` (background, text, button colors)
- Haptic feedback: `impactOccurred('light')` on answer tap, `notificationOccurred('success')` on correct, `notificationOccurred('error')` on wrong
- `Telegram.WebApp.ready()` called on load
- `Telegram.WebApp.expand()` called to use full screen
- Graceful degradation: all of the above wrapped in availability checks, game works fine without it

**Deliverables**:
- Telegram integration layer in `index.html`
